> 本文档同时是**计划书 + 进度追踪器**。生成一篇，标记一篇 ✅。

## 1. 篇幅规则（与 STRATEGY.md §2 表E 同步）

| 章节范围 | 比例 | 说明 |
|:--------|:---:|------|
| Cap.1-4 | 100% 短篇（150-349词） | 词汇 < 1500 |
| Cap.5-9 | 短 : 中 = **5 : 3** | 长篇不做 |
| Cap.10+ | 短 : 中 : 长 = **5 : 3 : 2** | 长篇最后统一做 |

## 2. 管线（一条命令生成一篇）

### 核心

```bash
# Step 1 — 生成提示词
python AI_reader/generate_prompt.py --chapter N --vocab --history

# Step 2 — AI 输出后，合并评估写入
python AI_reader/merge_yaml.py -f <ai_output_file>
```

### 可选（按需）

| 脚本 | 用途 |
|:---|:---|
| `verify_vocab.py` | 验证单篇词汇合规 |
| `reaudit_all_stories.py` | 全量审计 |
| `check_duplicates.py` | 标题重复扫描 |

## 6. 执行流程

```

3. AI 阅读 STRATEGY.md + 提示词 → 输出故事
4. python AI_reader/merge_yaml.py -f <output> → 自动写入 realitates/
5. 验证: 检查 v2_level ≤ target+2
6. 更新本文档: 标 ✅
```
