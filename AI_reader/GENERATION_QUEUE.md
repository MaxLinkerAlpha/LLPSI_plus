## 2. 实际管线（一条命令生成一篇）

### 核心（必用）

```bash
# Step 1 — 生成提示词（注：AI实际阅读STRATEGY.md后自行构造，不跑脚本）
# 如需严格词表模式（Cap.1-10推荐）：
python AI_reader/generate_prompt.py --chapter N --vocab --history

# Step 2 — AI 输出后，合并评估并写入文件
python AI_reader/merge_yaml.py -f <ai_output_file>
```

### 可选（按需）

| 脚本 | 用途 |
|:---|:---|
| `verify_vocab.py` | 验证单篇词汇合规 |
| `reaudit_all_stories.py` | 全量审计所有故事 |
| `check_duplicates.py` | 标题重复扫描 |
| `_scan_dist.py` | 重新扫描分布（修改队列后刷新） |

> 注：`progress.json` / `realitates.json` 由 `merge_yaml.py` 自动维护，**不用于进度追踪**。进度直接看本文档标记。

## 6. 执行流程（每章）

```
1. 读取本文档当前章清单
2. python AI_reader/generate_prompt.py --chapter N [--vocab] [--history]
3. AI 阅读 STRATEGY.md + 提示词 → 输出 N 篇
4. python AI_reader/merge_yaml.py -f <output>  → 自动写入 realitates/
5. 验证: 检查 v2_level ≤ target+2
6. 更新本文档: 将末尾 x{N} 拆成 {N} 行，每行 ✅/❌
```
