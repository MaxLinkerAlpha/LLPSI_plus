## 2. 两种生成模式（按章节分叉）

### 模式 A：自由生成 + 算法分配（Cap.10+）

**原理**：词汇充裕（3600+ 词形），AI 自由发挥命中率 > 60%。不限制词表，让 AI 写好故事，算法自动把故事归到正确的章节。

```bash
# Step 1 — 生成提示词（不加 --vocab，不限制词表）
python AI_reader/generate_prompt.py --chapter N --history

# Step 2 — AI 输出后，合并评估写入
python AI_reader/merge_yaml.py -f <ai_output_file>

# Step 3 — 验证双重门槛（来自难度算法 v3）
#   · 覆盖率 ≥ 85% → 可评级（否则降级，不给虚假章节号）
#   · v2_level ≤ target+2 → 通过（否则故事太难）
```

> 参考：`DIFFICULTY_ALGORITHM.md` §三 Step 2（收录率阈值 + 80th 百分位双重门槛）

### 模式 B：方案D 严词表（Cap.1-10）

**原理**：词汇 < 3600 词形，自由命中率仅 4%（Cap.1-5 实测）。必须用严格词表 + 模板扩充。命中率 100%（Cap.1-5 实战：53/53 通过）。

```bash
# Step 1 — 加载本章累计词表
python AI_reader/generate_prompt.py --chapter N --vocab --history
# --vocab 会注入 vocab_by_chapter/cap{N}_vocab_clean.json 的全部可用词形

# Step 2 — AI 阅读提示词 + 词表 → 严格用词表内词汇写作 → 输出故事
#   方案D 黑名单见 Lesson.md L04（动词/名词/专名禁止项）
#   篇幅不足用 5 套模板：重复/排比/问答/反转/感官细节

# Step 3 — 验证
python AI_reader/merge_yaml.py -f <ai_output_file>
#   检查：覆盖率 ≥ 85% 且 v2_level ≤ target+2

# Step 4 — 失败则反馈 OOV 词，回到 Step 2 重写
```

> 参考：STRATEGY.md §1.2 L44-56 + `.trae/skills/vocab-constrained-story.md`

## 8. 算法结构性限制分析（v2_3_0 新增）

### 8.1 核心问题

`evaluate_v2.py` 使用 **80 分位数 + 85% 覆盖率** 双重门槛定级。这对 Cap.5-30 运作良好，但对边缘章节（Cap.1-4 和 Cap.31-35）存在结构性限制。

### 8.2 Cap.2-4：低章定级困境

**症状**：即使写极简故事（如「Puer et avis」，80 词），80 分位数仍被推到 Cap.10。

**根因**：
- `form_chapter_map.json` 中 Cap.1-2 可用词汇极少（597 词形，含大量 OCR 碎片如 "ancil"、"capitul"）
- 极简故事中使用的词汇（avis→Cap.7、pulchra→Cap.5、arbore→Cap.5）在 LLPSI 中首次出现于 Cap.5-7
- 常用词占比过高（"est"、"et"、"in" 等来自 Cap.1-2，但 "avis"、"pulchra" 等语义词来自 Cap.5-7）

**实测数据**（batch21 的「Puer et avis」）：
| 章节范围 | 词数 | 占比 |
|:--------|:---:|:---:|
| Cap.1-2 | 32 | 47.1% |
| Cap.3-4 | 7 | 10.3% |
| Cap.5-6 | 6 | 8.8% |
| Cap.7-10 | 23 | 33.8% |
| **80 分位数** | | **Cap.10** |

### 8.3 Cap.34-35：高章定级困境

**症状**：即使密集使用语法术语（nōminātīvus、genetīvus、coniūnctiō），80 分位数仍停在 Cap.17-18。

**根因**：
- 语法术语在 LLPSI 中首次出现于早期章节：nōminātīvus→Cap.3、genetīvus→Cap.2、coniūnctiō→Cap.32、praepositiō→Cap.6
- `form_chapter_map.json` 中 Cap.35 的 1314 个词形大量是 OCR 拼接碎片（如 "iungereiūnxisseiūnctum"、"commiūniterve"、"grammaticaDonātus"）
- 任何拉丁文本中常用词（est、et、in、nōn、sed、sum、ego）占 30-40%，全部来自 Cap.1-2
- 即使 60% 的词来自 Cap.34-35，仍有 22%+ 的词在 `form_chapter_map.json` 中找不到（如 "discipulōs"、"scholā"、"flectitur"），导致覆盖率 < 85%

**实测数据**（batch23 的「Dēclīnātiōnis memorābilis」）：
- 80 分位数：Cap.35（表面达标）
- 但覆盖率：仅 77%（22/95 词未收录），低于 85% 阈值 → 被降级

### 8.4 可选解决方案

| 方案 | 描述 | 工作量 | 风险 |
|:---:|------|:---:|------|
| A. 手动直写 | 绕过 merge_yaml.py，直接写 .md 文件到 realitates/CapN/，手动设置元数据 | 低（~1h） | 故事没有算法验证的难度标签 |
| B. 修改算法 | 对 Cap.1-5 用中位数替代 80 分位数；对 Cap.30-35 放宽覆盖率到 75% | 中（~2h） | 可能影响其他章节的定级准确性 |
| C. 扩充词表 | 清洗 form_chapter_map.json 中的 OCR 碎片，补充缺失词形 | 高（~4h） | 需要人工校对 OCR 输出 |
| D. 接受现状 | Cap.34-35 保持空白，Cap.2-4 保持当前数量 | 零 | 不满足计划目标 |

> **推荐方案 A**：最快速，保证每章至少 10 篇。故事质量由人工 review 保证，不依赖算法验证。
