# LLPSI Plus

## 项目缘起 —— Why？

### 核心矛盾

教材进阶节奏过快，单章节难度过高难以通读

## 快速简介 —— What？

### 主要

对LLPSI每章节进行难度分级，根据难度匹配拓展读物（N+1），避免出现卡章的情况

### 衍生

LLPSI 电子化，一个网页读全系列

## 项目解决思路 —— How？

### 面向实现

#### 难度计算

- 以LLPSI每章难度为标准
- 电子化LLPSI：计算文本难度，自动按难度拆分读物章节（整本难度相近则不拆分），匹配 N+1 读者水平
- 必须先用带长音的OCR获取OCR文本

#### 自动匹配

上传读物自动 OCR、分板块，将相应板块匹配至对应学习章节——或者说至少得出一个难度评级

### 面向普通用户

- **使用方式 1**：选定学习到的章节，查看该章节适配拓展读物
- **使用方式 2**：批量导入自有文本材料，系统自动OCR、评估难度并精准匹配对应学习章节
- **使用方式3**：输入当前掌握水平，跳转相应LLPSI章节

## 项目里程碑—— 具体怎么解决？

> **核心原则**：遵循 LLPSI 教学法，拉丁释义拉丁、语境推导、N+1 循序渐进、带长音
